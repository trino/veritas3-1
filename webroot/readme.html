  • Returned data
    • The system will return a JSON object with "Status" being either true or false
    • If "Status" is true, the "OrderID" variable will give the Order ID number that was made, that can then be used to check the order status
    • If "Status" is false, the "Reason" variable will indicate why
      • %email% is in use
        • That email is in use and cannot be reused
      • %field% is required and missing
        • The field is required and missing
      • Username not specified
        • username is missing
      • Username '%username%' not found
        • The username specified was not found in the database
      • Password mismatch
        • The password specified is incorrect or not MD5'd
      • %Field% (%Value%) is not a valid %Type%
        • The value specied for the field, is not a valid value for that field's data type
      • Not a valid client ID
        • The clientid was either not specified or not found in the database
  • Check Order status
    • if you specify the "action" primary field as "orderstatus", you can check the status of order placed by yourself
    • POST data: (* denotes required fields)
      • username string* Your username
        password string* Your MD5'd Password
        action string* ["orderstatus"]
        orderid number* ID of the order placed by yourself
        pretty N/A If present, the JSON will be outputted in "pretty" format
    • Returned data:
      • The system will return a JSON object with "Status" being either true or false
      • If "Status" is true, the order status will be returned with the following fields
        • "id", "user_id", "client_id", "uploaded_for", "created", "ins_id", "ebs_id", "ins_79", "ins_79_binary", "ins_1", "ins_1_binary", "ins_14", "ins_14_binary", "ins_77", "ins_77_binary", "ins_78", "ins_78_binary", "ebs_1603", "ebs_1603_binary", "ebs_1627", "ebs_1627_binary", "ebs_1650", "ebs_1650_binary", "ins_72", "ins_72_binary", "order_type", "forms", "ins_31", "ins_31_binary", "ins_32", "ins_32_binary"
        • "complete"
          • Once all 3 products have been returned, the complete field will be marked as 1.
        • "Files"
          • An array of complete URLs to the files pertaining to this order
      • If "Status" if false, the "Reason" variable will indicate what error occured:
      • Username not specified/Username '%username%' not found/Password mismatch
      • Order not found
        • The Order ID specified could not be found
      • You did not place this order
        • The order was not placed by whoever was specified with the username/password combination, and cannot be checked

    • function array_flatten($array){
	if (isset($array["form"])) {
		foreach ($array["form"] as $ID => $Data) {
			foreach ($Data as $Key => $Value) {
				$array["data[" . $ID . "][" . $Key . ']'] = $Value;
			}
		}
	}
	unset($array["MAX_FILE_SIZE"]);//not needed
	unset($array["form"]);
	return $array;
}
